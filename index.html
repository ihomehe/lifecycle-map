03 · Execution Layer

Example campaign logic, QA checkpoints, and sample messaging

This layer translates lifecycle strategy into execution-ready CRM touchpoints. Each example includes channel, trigger, message intent, and a practical QA lens for deployment.

Channel Trigger Sample Message Execution / QA Check
Push Day 1 after install if no journal has been recorded "Try a 60-second check-in. Your space is private, and you can start small." Confirm trigger suppresses after first journal entry; verify deep link opens recording screen correctly.
Email Day 3 after install if onboarding remains incomplete Subject: "Your space is ready when you are"
Body intent: reduce pressure, reinforce privacy, and guide first meaningful action.		 Check subject line rendering, CTA link accuracy, mobile layout, and send suppression for activated users.
Email + Push 7 days without journal entry or coping session "It’s been a while. Even 60 seconds can help · we kept your space warm." Validate inactivity window, deduplicate cross-channel sends, and confirm message stops after return activity.
Email 30 days inactive plus new podcast episode publish Subject: "A new episode just dropped"
Body intent: reconnect through resonance rather than urgency.		 Confirm content block populates latest episode correctly, links resolve, and segmentation excludes recently reactivated users.
